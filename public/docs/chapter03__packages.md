# 3.2. Todos os Pacotes

### Nota

Leia os avisos de segurança antes de baixar os pacotes para descobrir se uma versão mais recente de qualquer pacote deve ser usada para evitar vulnerabilidades de segurança.

As fontes upstream podem remover versões antigas, especialmente quando essas versões contêm uma vulnerabilidade de segurança. Se uma URL abaixo não estiver acessível, você deve ler os avisos de segurança primeiro para descobrir se uma versão mais recente (com a vulnerabilidade corrigida) deve ser usada. Caso contrário, tente baixar o pacote removido de um mirror. Embora seja possível baixar uma versão antiga de um mirror mesmo que esta versão tenha sido removida devido a uma vulnerabilidade, não é uma boa ideia usar uma versão conhecida por ser vulnerável ao construir seu sistema.

### Nota

O kernel Linux é atualizado com bastante frequência, muitas vezes devido a descobertas de vulnerabilidades de segurança. A versão estável mais recente do kernel disponível pode ser usada, a menos que a página de erratas indique o contrário.

Para usuários com velocidade limitada ou largura de banda cara que desejam atualizar o kernel Linux, uma versão base do package e patches pode ser baixada separadamente. Isso pode economizar algum tempo ou custo para uma atualização de nível de patch subsequente dentro de uma versão menor.
