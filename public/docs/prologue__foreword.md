# i. Prefácio

Minha jornada para aprender e entender melhor o Linux começou em 1998. Eu tinha acabado de instalar minha primeira distribuição Linux e rapidamente fiquei intrigado com todo o conceito e filosofia por trás do Linux.

Sempre há muitas maneiras de realizar uma única tarefa. O mesmo pode ser dito sobre as distribuições Linux. Muitas existiram ao longo dos anos. Algumas ainda existem, algumas se transformaram em outra coisa, enquanto outras foram relegadas às nossas memórias. Todas fazem as coisas de forma diferente para atender às necessidades de seu público-alvo. Como existem tantas maneiras diferentes de alcançar o mesmo objetivo final, comecei a perceber que não precisava mais ser limitado por uma única implementação. Antes de descobrir o Linux, nós simplesmente tolerávamos problemas em outros Sistemas Operacionais, pois não havia escolha. Era o que era, gostando ou não. Com o Linux, o conceito de escolha começou a surgir. Se você não gostava de algo, era livre, e até encorajado, a mudá-lo.

Experimentei várias distribuições e não consegui me decidir por nenhuma. Eram ótimos sistemas por si só. Não era mais uma questão de certo ou errado. Tinha se tornado uma questão de gosto pessoal. Com toda essa escolha disponível, ficou evidente que não haveria um único sistema que seria perfeito para mim. Então, decidi criar meu próprio sistema Linux que se conformaria totalmente às minhas preferências pessoais.

Para realmente torná-lo meu próprio sistema, resolvi compilar tudo a partir do código-fonte em vez de usar pacotes binários pré-compilados. Este sistema Linux “perfeito” teria os pontos fortes de vários sistemas sem suas fraquezas percebidas. A princípio, a ideia era bastante assustadora. Permaneci comprometido com a ideia de que tal sistema poderia ser construído.

Depois de resolver problemas como dependências circulares e erros de tempo de compilação, finalmente construí um sistema Linux personalizado. Estava totalmente operacional e perfeitamente utilizável como qualquer outro sistema Linux existente na época. Mas era minha própria criação. Foi muito gratificante ter montado um sistema assim por conta própria. A única coisa melhor teria sido criar cada peça de software por conta própria. Esta foi a segunda melhor coisa.

À medida que compartilhava meus objetivos e experiências com outros membros da comunidade Linux, ficou evidente que havia um interesse sustentado nessas ideias. Rapidamente ficou claro que tais sistemas Linux personalizados servem não apenas para atender a requisitos específicos do usuário, mas também como uma oportunidade de aprendizado ideal para programadores e administradores de sistema aprimorarem suas habilidades (existentes) em Linux. Desse interesse ampliado, nasceu o Projeto Linux From Scratch.

Este livro Linux From Scratch é o núcleo central desse projeto. Ele fornece o contexto e as instruções necessárias para você projetar e construir seu próprio sistema. Embora este livro forneça um template que resultará em um sistema funcionando corretamente, você está livre para alterar as instruções para se adequar a você, o que é, em parte, uma parte importante deste projeto. Você permanece no controle; nós apenas damos uma ajuda para você começar sua própria jornada.

Espero sinceramente que você se divirta muito trabalhando em seu próprio sistema Linux From Scratch e aproveite os inúmeros benefícios de ter um sistema que é verdadeiramente seu.
